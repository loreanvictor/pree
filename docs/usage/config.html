--- title: "pree: config" layout: ../_layout.html --- # Configuration To configure `pree`, either provide params and options to the CLI: ```bash pree build docs site -b repo ``` Or provide a config file: ```yaml # .pree.yml src: docs dest: site base: repo # ðŸ–• running `pree build` with this config file # is equivalent to running the previous command. ```
A config file is normally a [YAML file](https://yaml.org/) named `.pree.yml` located at working directory, but its name and address can also be configured. The config file is treated as a convenient default, any options provided to the CLI will override the config file.
Config & CLI Options
Config CLI Description
`src` _first param_ The source of website content.
Either an HTML file or a directory.
CLI Example ```bash pree view docs ```
Config Example ```yaml src: docs ```
`dest` _second param_ The destination of the built website.
Either an HTML file or a directory.
CLI Example ```bash pree build docs site ```
Config Example ```yaml src: docs dest: site ```
`base` `-b`, `--base` The [base path](usage/index#asset-management) of the website.
CLI Example ```bash pree build docs site -b repo ```
Config Example ```yaml base: repo ```
`include` `-i`, `--include` A [glob pattern](https://en.wikipedia.org/wiki/Glob_(programming)) to include files for build.
> For multiple patterns, use multiple `-i` flags,
or provide an array in the config.
CLI Example ```bash pree build -i "**/*.final.html" ```
Config Example ```yaml src: docs include: "**/*.final.html" ```
`exclude` `-e`, `--exclude` A [glob pattern](https://en.wikipedia.org/wiki/Glob_(programming)) to exclude files from build.
Defaults to `['**/_*', '**/.*']`.
> For multiple patterns, use multiple `-e` flags,
or provide an array in the config.
CLI Example ```bash pree build -e "**/*.draft.html" ```
Config Example ```yaml src: docs exclude: "**/*.draft.html" ```
`port` `-p`, `--port` The port to serve the website on.
Defaults to `3000`.
CLI Example ```bash pree view -p 8080 ```
Config Example ```yaml port: 8080 ```
`prod` `-P`, `--prod` View the website in production mode.
Disables [build env access](components/build-env).
CLI Example ```bash pree view -P ```
Config Example ```yaml prod: true ```
`concurrency` `-C`, `--concurrency` The number of concurrent pages to build at once.
Defaults to `16`.
CLI Example ```bash pree build -C 8 ```
Config Example ```yaml concurrency: 8 ```
`root` `-r`, `--root` The root path of the website.
Defaults to `src`.
CLI Example ```bash pree view -r path ```
Config Example ```yaml root: path ```
`logLevel` The log level of the CLI.
One of `debug`, `info`, `warn`, `error`, `silent`.
Defaults to `info`.
Config Example ```yaml logLevel: debug ```
`-c`, `--config` The path to the config file.
Defaults to `.pree.yml`.
CLI Example ```bash pree build -c preeconf.yml ```
`-V`, `--verbose` Show verbose output.
CLI Example ```bash pree build -V ```
`-S`, `--silent` Silent mode.
Use multiple times to silence more.
CLI Example ```bash pree build -SS ```
